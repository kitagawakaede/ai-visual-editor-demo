# ヘアスタイル診断機能 設計ドキュメント

> 作成日: 2026-07-01
> ブランチ: `feature/hairstyle-diagnosis`
> 対象: TEXTLENS 27SS 展示会デモアプリ（6個目の機能）

## 1. 目的

本人写真から髪型のみを変えた9パターンを並列生成し、そのうち上位3枚を
AI採点して「一番似合う／普通×2」として星評価付きで提示する、ヘアスタイル診断機能を追加する。
既存の TimeSlip 機能（9枚並列生成＋性別判定＋QR共有）とほぼ同型の構造で実装する。

## 2. 全体フロー

```
カメラ撮影（シャッター）
  → 「診断」ボタン
  → detectGenderFromImage で男女判定（man / woman）
  → 9枚を並列生成（本人の顔を保持し髪型のみ変更・参照画像なしのテキスト生成・性別出し分け）
      - 上位3枚（featured）
      - その他6枚（others）
  → 上位3枚のみ scoreHairstyle で AI採点（2項目・各1〜5）
  → 合計スコア順にソート → 1位=「一番似合う」、2・3位=「普通」
  → 結果レイアウト表示（上段3枚比較カード + 下段6枚グリッド）
  → 9枚コラージュ合成 → Supabase アップロード → QR（後追い表示）
```

## 3. 生成する9枚（髪型リスト）

- **本人の顔・輪郭・骨格を保持し、髪型のみを変更する。**
- **参照画像は渡さない（テキストのみ生成）** — 別人の顔が混入せず本人を保てる（TimeSlip と同方針）。
- **性別で出し分け**（`promptByGender`）。男女それぞれ 上位3 + その他6 = 9枠を定義する。

### 女性（woman）
| 区分 | 髪型名 |
|------|--------|
| 上位3（採点） | くびれミディ×シースルーバング |
| 上位3（採点） | ナチュラルボブ×軽め外ハネ |
| 上位3（採点） | ゆる巻きセミロング |
| その他6 | 外ハネボブ |
| その他6 | 耳かけボブ |
| その他6 | ハーフアップ |
| その他6 | 低めポニー |
| その他6 | 韓国風ミディ |
| その他6 | センターパートロング |

### 男性（man）
| 区分 | 髪型名 |
|------|--------|
| 上位3（採点） | 爽やかマッシュ×シースルーバング |
| 上位3（採点） | ナチュラルショート×軽く流し |
| 上位3（採点） | センターパート×ゆる束感 |
| その他6 | ツーブロックショート |
| その他6 | アップバング |
| その他6 | 韓国風マッシュ |
| その他6 | ソフトモヒカン |
| その他6 | かき上げミディ |
| その他6 | パーマ束感ショート |

> 髪型名・文言は実装時に微調整可（先方フィードバック前提）。個数（上位3＋その他6＝9）は固定。

## 4. 星評価（上位3枚のみ・AI採点）

- **採点対象**: 上位3枚の「生成後の画像」を gpt-4o（vision）で採点する。その他6枚は採点しない。
- **評価項目（性別で異なる）**:
  - 女性: 小顔効果・垢抜け度
  - 男性: 爽やかさ・垢抜け度
- 各項目 **1〜5 の整数**（星の数）で返す。
- **ソート**: 3枚を「2項目の合計スコア」降順で並べ替える。
  - 1位 → ラベル「一番似合う」（王冠アイコン）
  - 2・3位 → ラベル「普通」
- 採点失敗・API エラー時のフォールバック: 妥当な既定値（例 一番似合う=5/5, 普通=4/4, 3/3）を割り当て、ソートせずスロット順のまま表示する。

## 5. 結果レイアウト（UIモックに準拠）

- **枠の背景は白**（ポラロイド風の白枠）。
- **上段**: 上位3枚を横並び比較カード。各カードに
  - カテゴリ見出し（「一番似合う」＝王冠付き／「普通」）
  - 髪型名
  - 星評価2行（項目名＋★×5）
- **下段**: その他6枚を 3列×2行グリッド。髪型名のみ表示（星なし）。
- 右上に QRコード（生成中は「QR生成中…」）とクローズ（✕）ボタン。
- 生成完了後にフェードイン表示（TimeSlip と同様）。
- 結果表示はカメラ枠（aspect 3/4）に収めず、モックのように縦に伸びる専用セクションとして描画する。

## 6. コンポーネント / ファイル構成

### 新規
- `src/modules/HairStyleModule.tsx`

  - TimeSlipModule.tsx を下敷きにするが、結果レイアウト（3枚比較＋6枚グリッド＋星）が異なる。

### 変更

  - `HairStyleItem` 型（`id` / `label` / `promptByGender` / `scored: boolean`）
  - `hairStyleBase(scene)` … ヘア用ベースプロンプト（本人の顔・輪郭を厳守し髪型のみ変更、参照画像なし前提の文面）

## 7. エンジン方針

- 既定 **Gemini**（速度重視・髪型変更は本人の顔保持が主眼で Gemini 向き）。
- `EngineToggle` で OpenAI へ切替可（切替時は9枚とも OpenAI で生成）。他機能と統一。
- 採点（`scoreHairstyle`）はエンジン切替に関わらず常に gpt-4o（OpenAI）。`VITE_OPENAI_API_KEY` 前提。
- QR用 Supabase は既存設定を流用。

## 8. エラーハンドリング

- 9枚並列生成は `Promise.all`。部分失敗した枠は「生成失敗」プレースホルダ表示（TimeSlip と同様）、揃った枚数だけ表示。
- 全滅時はエラー表示＋再生成ボタン。
- 採点失敗時はフォールバック星評価（第4章）。
- QR は後追い非同期。失敗しても本体表示は継続（`logError`）。

## 9. スコープ外 / 未決

- **タブアイコン素材**: `src/assets/UI/UI2/button/` は現状5個分のみ。6個目アイコンは別途素材が必要。**暫定で既存アイコン流用またはプレースホルダで実装**し、素材入手後に差し替える。
- 髪型名・プロンプト文言・星の既定値は先方フィードバックで微調整前提。
- 上位3枚以外（その他6枚）の採点は行わない（今回スコープ外）。
