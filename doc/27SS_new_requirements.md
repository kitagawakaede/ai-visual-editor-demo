## 1. LINEスタンプ

### 1-1. バックエンド（生成ロジック）

- **やりたいこと**: 6枚のスタンプが**グリッド形式で1枚に並んだ画像**を **一括で1枚** 生成する。
  - 現状は1テキスト＝1スタンプを個別生成（`STAMP_PROMPT_MAP` / `LINE_STAMP_PROMPT_BASE`、個別参照画像 `STAMP_REF_URLS`）。
  - 今回はそれとは別に「6枚グリッドを一括生成する」プロンプトを **新規追加** する。
- **参照イメージ（生成したい完成形）**: [`src/assets/スタンプ/image copy 12.png`](../src/assets/スタンプ/image%20copy%2012.png)
  - 内容: 3列×2行で6種のちびキャラスタンプが並ぶ。各スタンプに日本語テキスト（例: ヤッホー！／おはよー／よろしく／了解！／OK！／おつかれ）と白フチ付き。
- **既存プロンプトの扱い（重要）**:
  - [`src/constants/prompts.ts`](../src/constants/prompts.ts) の `stampBase` / `STAMP_PROMPT_MAP` / `LINE_STAMP_PROMPT_BASE` は **削除しない**。
  - これらが参考にしていた個別参照画像（例: [`src/assets/スタンプ/image copy 11.png`](../src/assets/スタンプ/image%20copy%2011.png) など）も残す。
  - **対応方針**: LINEスタンプ用プロンプトを **一種類だけ追加**（例: `LINE_STAMP_GRID_PROMPT`）。6枚グリッド一括生成用の定数を新設する。

### 1-2. フロントエンド（UI）

- **やりたいこと**: 添付の **LINEスタンプ購入画面を再現**する。
  - 参照: 本要望に添付されたスクリーンショット（LINEスタンプショップの購入画面）。
  - 主要素:
    - 上部: 戻る `<` / 共有アイコン / 閉じる `×`
    - 中央上: QRコード（`MY STAMP` ラベル）
    - アクション行: ハート（`10,000`）／`プレゼントする`／`購入する`（緑ボタン）
    - 下部: 生成した6枚スタンプのグリッド表示
- **表示位置**: **カメラの枠内**に収めて表示する。
- 現状フロント [`src/modules/LineStampModule.tsx`](../src/modules/LineStampModule.tsx) は単体スタンプ表示。購入画面UIへ差し替え/拡張する。

## 5. 決定事項（2026-06-26 確認済み）

### LINEスタンプ（バックエンド）
- 6枚のテキスト/ポーズは **一旦固定**（見本 `image copy 12.png` の「ヤッホー！／おはよー／よろしく／了解！／OK！／おつかれ」。今後変わる可能性あり）
- グリッド生成プロンプトの**2枚目参照画像は `image copy 12.png`** を使用
- フロントは購入画面に**置き換え**。旧・個別生成UIは廃止し、**プロンプト（`STAMP_PROMPT_MAP` 等）はコードとして残す**
- 生成画像レイアウトは **2列 × 3行の縦長グリッド**（→ フロントでスクロール表示）

### LINEスタンプ（フロント・購入画面）
- QR（MY STAMP）は **生成した6枚グリッド画像（Supabase）へのリンク**
- `プレゼントする`／`購入する`／ハート(10,000) は **ダミー表示**（押下しても何もしない）
- スタンプ表示部分は **2列縦長グリッド**で、**スタンプ部分のみスクロール**可能にする

### タイムスリップ
- 9枚並列は **OpenAI gpt-image-1.5（2画像入力）** で生成
- 性別の出し分け: 小学生は男女で服を変える。それ以外（昭和レトロ等の女性参照）は **それなりに自然に寄せる**（撮影者が男性でも破綻しない範囲で）

### 全体
- 9枚並列のタイムスリップでも、撮影後の**待機ゲームを表示する**
- **全体UI修正は各機能の実装完了後**に実施
