# ヘアスタイル診断機能 設計ドキュメント

> 作成日: 2026-07-01
> ブランチ: `feature/hairstyle-diagnosis`
> 対象: TEXTLENS 27SS 展示会デモアプリ（6個目の機能）

## 1. 目的

本人写真から髪型のみを変えた9パターンを並列生成し、そのうち上位3枚を
AI採点して「一番似合う／普通×2」として星評価付きで提示する、ヘアスタイル診断機能を追加する。
既存の TimeSlip 機能（9枚並列生成＋性別判定＋QR共有）とほぼ同型の構造で実装する。

## 2. 全体フロー

```
カメラ撮影（シャッター）
  → 「診断」ボタン
  → detectGenderFromImage で男女判定（man / woman）
  → 9枚を並列生成（本人の顔を保持し髪型のみ変更・参照画像なしのテキスト生成・性別出し分け）
  → 9枚すべてを scoreHairstyle で AI採点（Promise.all で並列。2項目・各1〜5）
  → 合計スコア順に全9枚をソート
      - 上位3枚 → 比較カード（星付き）。1位=「一番似合う」、2・3位=「普通」
      - 残り6枚 → グリッド（星なし）
  → 結果レイアウト表示（上段3枚比較カード + 下段6枚グリッド）
  → 9枚コラージュ合成 → Supabase アップロード → QR（後追い表示）
```

## 3. 生成する9枚（髪型リスト）

- **本人の顔・輪郭・骨格を保持し、髪型のみを変更する。**
- **参照画像は渡さない（テキストのみ生成）** — 別人の顔が混入せず本人を保てる（TimeSlip と同方針）。
- **性別で出し分け**（`promptByGender`）。男女それぞれ 9枠を定義する。
- 9枠はすべて対等な候補。**採点結果の上位3枚が「比較カード（星付き）」に昇格**し、残り6枚が「その他グリッド」になる（区分は固定ではなくスコアで決まる）。

## 4. 星評価（9枚すべてをAI採点 → 上位3枚を昇格）

- **採点対象**: 生成に成功した**9枚すべて**の「生成後の画像」を gpt-4o（vision）で採点する。
- **並列実行**: 採点は `Promise.all` で全枚同時に投げる。3枚→9枚に増えても待ち時間はほぼ変わらない（同時並列のため wall-clock は概ね一定）。生成9枚とは別工程。
- **評価項目（性別で異なる）**:
  - 女性: 小顔効果・垢抜け度
  - 男性: 爽やかさ・垢抜け度
- 各項目 **1〜5 の整数**（星の数）で返す。
- **ソート**: 生成成功した全枚を「2項目の合計スコア」降順で並べ替える。
  - 上位3枚 → 比較カード（星付き）。1位=「一番似合う」（王冠）、2・3位=「普通」
  - 残り6枚 → グリッド（星なし）
- 採点失敗・API エラー時のフォールバック: `{small:3, refined:3}` を割り当てる（例外は投げない）。全枚同点になった場合はスロット順で上位3枚を選ぶ。

## 5. 結果レイアウト（UIモックに準拠）

- **枠の背景は白**（ポラロイド風の白枠）。
- **上段**: 上位3枚を横並び比較カード。各カードに
  - カテゴリ見出し（「一番似合う」＝王冠付き／「普通」）
  - 髪型名
  - 星評価2行（項目名＋★×5）
- **下段**: その他6枚を 3列×2行グリッド。髪型名のみ表示（星なし）。
- 右上に QRコード（生成中は「QR生成中…」）とクローズ（✕）ボタン。
- 生成完了後にフェードイン表示（TimeSlip と同様）。
- **結果は他機能（TimeSlip 等）と同じくカメラ枠（aspect 3/4）内のオーバーレイとして表示する**（ヘッダに「診断結果」＋QR＋✕、本文に上位3カード＋6グリッド。枠内に収まらない場合は縦スクロール）。

## 7. エンジン方針

- 既定 **Gemini**（速度重視・髪型変更は本人の顔保持が主眼で Gemini 向き）。
- `EngineToggle` で OpenAI へ切替可（切替時は9枚とも OpenAI で生成）。他機能と統一。
- 採点（`scoreHairstyle`）はエンジン切替に関わらず常に gpt-4o（OpenAI）。`VITE_OPENAI_API_KEY` 前提。
- QR用 Supabase は既存設定を流用。

## 8. エラーハンドリング

- 9枚並列生成は `Promise.all`。部分失敗した枠は「生成失敗」プレースホルダ表示（TimeSlip と同様）、揃った枚数だけ表示。
- 全滅時はエラー表示＋再生成ボタン。
- 採点失敗時はフォールバック星評価（第4章）。
- QR は後追い非同期。失敗しても本体表示は継続（`logError`）。

## 9. スコープ外 / 未決

- **タブアイコン素材**: 6個目アイコンは `src/assets/髪型/image.png` を一旦使用する（差し替えの可能性あり）。
- 髪型名・プロンプト文言・星の既定値は先方フィードバックで微調整前提。
- その他6枚（グリッド）は採点はするが星は表示しない（モック準拠）。将来的に表示切替の余地あり。
